Duplicate DR Database Fix Test

This test verifies that the database UNIQUE constraint issue is resolved and multiple items with the same DR number can be saved.

Database Schema Update Required

Before running this test, you need to apply one of these SQL scripts in your Supabase SQL Editor:

  • Option 1 (Recommended): supabase/remove-dr-unique-constraint.sql
  • Option 2 (Alternative): supabase/add-composite-unique-constraint.sql
Test Scenario

Problem: Database UNIQUE constraint on dr_number prevents saving multiple items with same DR

Test Data:

  • DR-001 with Serial SN-001
  • DR-001 with Serial SN-002
  • DR-001 with Serial SN-003
  • DR-001 with Serial SN-004
Expected Result

Before Fix: Only 1 item saved, others fail due to UNIQUE constraint

After Fix: All 4 items saved successfully

Test Results
Database Schema Solutions
Option 1: Remove UNIQUE Constraint (Recommended)
-- Remove UNIQUE constraint from dr_number
ALTER TABLE public.deliveries 
DROP CONSTRAINT IF EXISTS deliveries_dr_number_key;
Option 2: Composite UNIQUE Constraint
-- Add composite unique on dr_number + serial_number
ALTER TABLE public.deliveries 
ADD CONSTRAINT deliveries_dr_serial_unique 
UNIQUE (dr_number, serial_number);
